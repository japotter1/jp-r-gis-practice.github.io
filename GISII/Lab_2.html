Question 2

# Plot countries, categorized by continent
plot(world["continent"], reset = FALSE)

# Set the radius of population symbols
cex <- sqrt(world$pop) / 10000

# Create centroids
world_cents <- st_centroid(world, of_largest_polygon = TRUE)
## Warning in st_centroid.sf(world, of_largest_polygon = TRUE): st_centroid assumes
## attributes are constant over geometries of x
# Add centroids on top of countries
plot(st_geometry(world_cents), add = TRUE, cex = cex)

Two similarities:

  1. Both categorize countries by continent using color
  2. Both represent each country’s population by the size of the centroid dots

Two differences:

  1. The plot in the book includes graticule lines, which can be created with st_graticule()
  2. The plot in the book projects the continents, centroids, and graticule lines into a different CRS (referred to as "+proj=eck4")

The cex argument controls the symbol size. Because the area of a circle is proportional to the square of its radius, and cex, I would assume, controls the radius of the symbol, it is important to assign cex to a value proportional to the square root of the population, in order for the area to be directly proportional to the population. It was then divided by 10000 because, otherwise, the ciricles would be way too large (as population numbers of countries are high, and graphing units are generally much smaller).

By simply typing the name of the object, in this case nlcd, we can find:

  • The object’s class (the data format)
  • Dimensions (number of rows, columns, and layers)
  • Resolution (size of each cell)
  • Extent (spatial extent of whole raster)
  • The CRS (how the data are projected)
  • The source (where the file came from)
  • The name(s) of the layer(s)
  • The minimum and maximum values (in this case, I believe they are ordered alphabetically, as the data are categorical)
